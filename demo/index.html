Chapter 2: Understanding Page Capture

Page capture involves taking a screenshot of the visible portion of a web page. Modern browsers provide APIs that allow extensions to capture the current state of a tab, including all rendered content.

Diagram 2.1 - Capture Process

The Capture Process

When you initiate a capture, the extension performs several steps: first, it triggers the page navigation (via keyboard, click, or scroll), then it waits for the page to be fully loaded, and finally it captures the visible content.

Quality settings affect both the resolution of the capture and the compression of the resulting image. Higher quality means larger file sizes but better detail preservation.

Chapter 3: Handling Dynamic Content

Many modern web pages load content dynamically using JavaScript. This presents a challenge for automated capture because the page might appear ready before all content has actually loaded.

Readiness Detection

The extension uses several techniques to detect when a page is truly ready:

1. Document ready state checking - ensures the HTML has been parsed

2. Image loading detection - waits for images in the viewport to load

3. DOM stability monitoring - uses MutationObserver to detect when changes stop

Chapter 4: Duplicate Detection

Sometimes page navigation fails, resulting in the same page being captured multiple times. The extension uses perceptual hashing to detect these duplicates and avoid wasting resources.

How dHash Works

The difference hash (dHash) algorithm creates a fingerprint of an image by comparing adjacent pixels. Even if an image has slight variations (like different compression), similar images will have similar hashes.

Algorithm Visualization

By comparing the Hamming distance between hashes, we can determine if two images are visually similar enough to be considered duplicates.

Chapter 5: PDF Generation

Once all pages have been captured, the extension combines them into a single PDF document. Each captured image becomes a page in the PDF.

Page Sizing Options

You can choose how pages are sized in the final PDF:

Auto: Each PDF page matches the dimensions of its captured image, preserving the original aspect ratio.

A4 or Letter: Images are scaled to fit within standard paper sizes, useful for printing.

Chapter 6: Best Practices

To get the best results from automated page capture, follow these guidelines:

Configuration Tips

Delay Settings: If pages load slowly, increase the delay between page turns. A longer delay ensures content is fully loaded before capture.

Wait Selectors: If pages have a specific element that appears when ready, use a wait selector to ensure that element exists before capturing.

Settings Guide

Quality Balance: Higher quality produces better images but larger files. Find the balance that works for your needs.

Chapter 7: Troubleshooting

Here are solutions to common issues you might encounter:

Blank Captures

If captures appear blank, the page might be using canvas rendering or have content protection. Try increasing the delay or check if the site is compatible.

Duplicate Detection Stops Too Early

If the extension stops capturing before reaching the end, your pages might be too similar. Try adjusting the duplicate tolerance setting.

Chapter 8: Advanced Features

The extension includes several advanced features for power users:

Crash Recovery

If your browser crashes during a capture session, the extension saves your progress. When you reopen the browser, you can resume from where you left off or build a partial PDF.

Custom Filename Templates

Use template variables to create meaningful filenames automatically. Variables like {title}, {date}, and {pageCount} are replaced with actual values.

Chapter 9: Upload Integration

For automated workflows, you can configure the extension to upload PDFs to a remote server after generation.

Setting Up Upload

In the options page, enable upload and configure your endpoint URL, authentication headers, and upload method (POST multipart or PUT raw).

The extension validates your endpoint when you save settings to ensure it's accessible.

Upload Configuration

Security Considerations

Authentication tokens are stored locally in your browser. Make sure to use HTTPS endpoints and keep your tokens secure.
